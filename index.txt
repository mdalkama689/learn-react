1. why we need react?
People realised it's harder to do DOM manipulation the conventional way
There were libraries that came into the picture that made it slightly easy, but still for a very big app it's very hard (JQuery) Eventually, VueJS/React created a new syntax to do frontends
Under the hood, the react compiler convert your code to HTML/CSS/JS


// simple counter app

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Click Counter</title>
</head>
<body style="display: flex; flex-direction: column; align-items: center; justify-content: center; height: 100vh; font-family: sans-serif; background-color: #f5f5f5;">

  <h1 style="margin-bottom: 20px;">Simple Counter</h1>

  <div style="display: flex; align-items: center; gap: 10px;">
    <button class="decrease-btn" style="padding: 8px 16px; cursor: pointer;">Decrease</button>
    <span class="counter-value" style="font-size: 1.5rem; min-width: 40px; text-align: center;">1</span>
    <button class="increase-btn" style="padding: 8px 16px; cursor: pointer;">Increase</button>
    <button class="reset-btn" style="padding: 8px 16px; cursor: pointer; background-color: #ff5555; color: white; border: none; border-radius: 4px;">Reset</button>
  </div>

  <script>
    const decreaseButton = document.querySelector(".decrease-btn");
    const increaseButton = document.querySelector(".increase-btn");
    const resetButton = document.querySelector(".reset-btn");
    const counterValueElement = document.querySelector(".counter-value");

    decreaseButton.addEventListener("click", () => {
      counterValueElement.textContent = Number(counterValueElement.textContent) - 1;
    });

    increaseButton.addEventListener("click", () => {
      counterValueElement.textContent = Number(counterValueElement.textContent) + 1;
    });

    resetButton.addEventListener("click", () => {
      counterValueElement.textContent = 0;
    });
  </script>
</body>
</html>

***Some react jargon***
To create a react app, you usually need to worry about two things
Creator of frontend frameworks realised that all website 
can effectively be divided intp two parts

a. State    b. Components

a. State/Re-rendring
An object that represents the current state of the app
It represents the dynamic things in your app (things that change)
For example, the value of the counter

** for our counter app we have state like this 
{
    counter: 1  // this is the state 
}
** In our html counter app all btns(increase, decrease and reset) is 
a component  it take state(currentCount)  as an input And is suppose 
to render accordingly.

** You usually have to define all your components once.
And then all yu have to do is update the state of your app.
React take care of re-rendring  your app.

Re-render means:
React runs your component function again to update the UI whenever
the state or props change.

üß† In simple words:
When your data (state) changes, React refreshes the part
of the screen that depends on that data.
It doesn‚Äôt reload the whole page, only updates the necessary DOM elements.



**React is a diff calculator/engine. it calculate diffrence between old state 
and new state and update it.
** State + Components = React Website 

** When the state chnages, React updates(re-renders) the part of the page that
depends on that state. Re-rendring means React changes the actual HTML on the screen
to match the new data/state. 

b.Components
A React component defines how a DOM element should render based 
on a given state. It is a reusable and dynamic HTML-like snippet
that automatically updates whenever the state changes.
